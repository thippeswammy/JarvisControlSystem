Build a system that works like this:

User Goal

Intent Understanding
↓
Context Fusion
↓
World-State Modeling
↓
Environment Perception
↓
Capability Discovery
↓
Tool Discovery
↓
Constraint & Safety Analysis
↓
Dynamic Planning
↓
Execution
↓
Verification
↓
Recovery / Retry
↓
Memory Update
↓
Continuous Learning


NOT:
- hardcoded apps
- hardcoded paths
- hardcoded selectors
- hardcoded workflows

The system should dynamically understand the environment and adapt automatically.

---

# Core Philosophy

The assistant should behave like an intelligent operating-system cognition layer.

Instead of:
- fixed automation
- static rules
- manual mappings

The system should:
- observe
- infer
- reason
- adapt
- verify

---

# Desired System Behavior

## Example

User:
"Open my first browser profile"

System should:
1. Detect active browsers
2. Detect available profiles/tabs/windows
3. Understand browser structure
4. Infer what "first profile" means
5. Focus existing instance if already open
6. Verify action success

WITHOUT:
- hardcoded browser names
- hardcoded paths
- fixed profile IDs
